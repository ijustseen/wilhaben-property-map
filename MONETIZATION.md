# Монетизация StudiWohnkarte

Документ описывает, как можно монетизировать проект, не ломая бесплатный доступ к карте для студентов.

## Продуктовая позиция

**StudiWohnkarte** — карта студенческого жилья в Австрии: квартиры (willhaben), WG (wg-gesucht), общежития (где есть каталог). Пользователь выбирает университет → видит жильё в городе кампуса, commute и избранное.

Сильные стороны для монетизации:

- Узкая аудитория с высокой мотивацией (сроки зачисления, ограниченный бюджет).
- Повторное использование в течение семестра (сравнение, алерты, избранное).
- Данные уже агрегируются из открытых источников — ценность в UX, фильтрах, commute и уведомлениях.

## Модель: freemium + B2B

### 1. Бесплатный уровень (Free)

Оставить щедрым, чтобы продукт рос органически и через вузы:

| Возможность | Free |
|-------------|------|
| Карты всех городов с университетами | ✓ |
| Фильтры, слои карты, commute к кампусу | ✓ |
| Аккаунт + безлимитное избранное | ✓ |
| Алерты по цене / новым объявлениям | ✗ |
| Сохранённые поиски | ✗ |
| Экспорт PDF | ✗ |

**Почему:** студенты ожидают бесплатный поиск жилья; paywall на просмотр карты убьёт рост. Избранное остаётся бесплатным — Plus продаёт алерты и экспорт в пик сезона (август–сентябрь).

### 2. StudiWohnkarte Plus (B2C подписка)

**Цена (ориентир):** €4.90/мес через [Lemon Squeezy](https://www.lemonsqueezy.com/). Только помесячно — для поиска квартиры годовая подписка не нужна.

| Возможность | Plus |
|-------------|------|
| Всё из Free (включая безлимитное избранное) | ✓ |
| До 5 сохранённых поисков | ✓ |
| Email-алерты: новые объявления по фильтрам | ✓ |
| Уведомления об изменении цены в избранном | ✓ |
| Сравнение commute к 2 кампусам | ✓ |
| Экспорт избранного + monthly cost (PDF) | ✓ |
| Приоритет в поддержке | ✓ |

**Оплата:** Lemon Squeezy Checkout (hosted). Переменные окружения:

- `LEMONSQUEEZY_PLUS_MONTHLY_URL`

**Техническая реализация (дорожная карта):**

1. **Сейчас:** страница `/pricing` с кнопками на checkout URL.
2. **Далее:** webhook Lemon Squeezy → `lib/subscriptions.ts` → поле `plan` в `data/users.json` или отдельный `data/subscriptions.json`.
3. **Далее:** middleware / API проверяет `plus` перед алертами и экспортом.

### 3. StudiWohnkarte Campus (B2B для вузов / ÖH)

Пакет для студсоветов, international office, ÖH:

- White-label виджет «Найти жильё рядом с [Uni]» на сайте вуза.
- Статистика спроса (средняя цена по PLZ, популярные районы) — анонимизированная.
- Слот «рекомендованные общежития / партнёры» на карте (платное размещение, чётко помечено).

**Цена:** €199–499/семестр за вуз или €999/год за сеть FH.

### 4. Партнёрские и рекламные потоки (осторожно)

| Канал | Риск | Рекомендация |
|-------|------|--------------|
| Affiliate willhaben / wg-gesucht | Низкий доход, ToS | Только если разрешено; иначе — ссылки без трекинга |
| Страховка (Haftpflicht), SIM, банки для студентов | Средний | Блок «Moving to Austria» на лендинге, не на карте |
| Риелторы / WG-агентства | Высокий (доверие) | Отдельная вкладка «Verified partners», не смешивать с органикой |

### 5. Данные и API (долгосрочно)

Агрегированные отчёты для городов/застройщиков (где спрос студентов, median rent по району) — только при достаточной базе пользователей и GDPR-compliance.

## Воронка

```
Лендинг → выбор университета → карта (free)
                ↓
        регистрация (избранное без лимита)
                ↓
   нужен alert / saved search / export → /pricing
                ↓
        Lemon Squeezy checkout → Plus
```

## Метрики успеха

- **Activation:** % пользователей, открывших карту после выбора uni.
- **Save rate:** сохранения на активного пользователя карты.
- **Conversion:** free → Plus (цель 2–5% в пик сезона).
- **Churn:** отмена после нахождения жилья (ожидаемо высокая — помесячная подписка).
- **B2B:** количество пилотов с ÖH / international office.

## Этические ограничения

- Не прятать объявления за paywall.
- Платные слоты всегда с бейджем «Partner».
- Не продавать персональные данные; алерты только opt-in.
- Open-source ядро карты можно оставить публичным; Plus = hosted alerts, sync, export.

## Следующие шаги разработки

1. Настроить продукт в Lemon Squeezy (monthly) и прописать checkout URL в `.env.local`.
2. Webhook `subscription_created` / `subscription_cancelled`.
3. UI upsell при попытке включить alert или saved search.
4. PDF export как первый «ощутимый» Plus-only feature.
